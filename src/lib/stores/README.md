# Stores & State Management (UTF-8 with BOM)

Ця директорія містить реактивний стан програми, реалізований за допомогою Svelte 5 Runes.

## 🏛️ Ключові модулі

1.  **`appState.svelte.ts` (Facade):** 
    *   Головна точка входу для UI-компонентів.
    *   Агрегує дані з інших сторів через функцію `getState()`.
    *   Містить глобальні обробники подій (клавіатура, миша).
2.  **`uiState.svelte.ts` (UI & Modals):**
    *   Керує станом видимості модальних вікон.
    *   Зберігає налаштування відображення (масштаб, вигляд карток, тема).
    *   Містить стан для `ActionConfirmationModal`.
3.  **`fileSystemState.svelte.ts` (Data Model):**
    *   Зберігає ієрархію папок, файлів та вкладок.
    *   Виконує CRUD операції над даними, викликаючи сервіси.
    *   Керує автоматичним призначенням гарячих клавіш.
4.  **`configState.svelte.ts` (Persisted Config):**
    *   Керує глобальними налаштуваннями програми.
    *   Синхронізує налаштування з `localStorage`.

## 🛠️ Патерни

*   **Runes:** Використовуйте `$state`, `$derived` та `$effect` для керування станом.
*   **Getters for Encapsulation:** Усі публічні властивості в об'єктах сторів реалізовані через гетери (наприклад, `get toastMessage()`), щоб запобігти прямому редагуванню стану ззовні без виклику спеціальних методів (Actions).
*   **Cross-Store Communication:** Стори можуть імпортувати та використовувати один одного безпосередньо.

## ⚠️ Правила
*   Не додавайте логіку в UI-компоненти. Уся бізнес-логіка має бути в `stores` або `services`.
*   При додаванні нового модального вікна, зареєструйте його стан у `uiState.svelte.ts` та додайте відповідні методи `open/close`.
