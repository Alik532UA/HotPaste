## Сенс нововведень

### Проблема
Зараз HotPaste дозволяє лише переглядати, копіювати, фільтрувати та редагувати існуючі сніпети. Але для повноцінної роботи як "Менеджера сніпетів", критично не вистачає можливостей керування: створення нових сніпетів прямо з UI, видалення, переміщення між вкладками, налаштування зовнішнього вигляду та зручного сортування на екрані.

### Рішення
1. **FAB (Кнопка +)** для миттєвого створення нової картки.
2. **Drag and Drop (DND)** механізм для візуального впорядкування карток.
3. **Контекстне меню** для кожної картки з розширеними діями (видалення, копіювання, переміщення).
4. **Модальне вікно "Налаштувати"** для кастомізації картки через UI (зміни вписуються в існуючий `_hotpaste.json`).

## Floating Action Button (Створення)

### UI компонента
- Кругла кнопка з іконкою `+` в правому нижньому куті екрану (з `position: fixed`, поверх всього інтерфейсу).
- Тінь, колір акценту, hover-анімація (збільшення).

### Поведінка (User Flow)
1. Користувач натискає `+`.
2. У сітці (CardGrid) **одразу** з'являється нова порожня картка.
3. Картка автоматично переходить у стан **Inline-редагування** (встановлюється фокус на textarea).
4. Файл фізично (на диску) ще **НЕ створюється**.
5. Якщо користувач натискає "Зберегти" (або `Ctrl+Enter`) і текст не порожній:
   - Файлу генерується ім'я (наприклад `snippet_YYYYMMDD_HHMMSS.txt` або `new_snippet.txt`).
   - Файл фізично записується на диск.
   - Картка стає звичайною.
6. Якщо користувач натискає "Скасувати" (або `Esc`): нова картка просто зникає з інтерфейсу.

## Зміна порядку карток (Drag & Drop)

### Механізм сортування
Порядок зберігається не зміною назв файлів, а у файлі конфігурації вкладки `_hotpaste.json`. 

**Оновлення структури `_hotpaste.json`:**
```json
{
  "tab": {
    "icon": "📂",
    "order": ["frequent.txt", "analyze-code.txt", "greeting.txt"]
  },
  "cards": { ... }
}
```

### Логіка відображення
- Якщо масив `tab.order` існує, картки сортуються згідно нього. 
- Файли, яких **немає** у масиві `order` (наприклад, щойно створені або додані ззовні), додаються **вкінець** (за алфавітом) після відсортованих.
- Після кожного перетягування (DragEnd) масив `tab.order` оновлюється та відбувається Debounced запис `_hotpaste.json`.

### Способи переміщення
1. **Drag-and-Drop (DND):** Затискання мишкою (через Svelte Native DND або HTML5 Drag API) — картка "відривається" від сітки та міняється місцями з іншими.
2. **Кнопкові дії:** З контекстного меню (Рухати → Вперед/Назад).

## Контекстне меню

### Виклик меню
- **Спосіб 1:** Клік правою кнопкою миші (Context Menu) по будь-якому місцю картки. Системне меню блокується (через `e.preventDefault()`).
- **Спосіб 2:** Клік лівою кнопкою миші на спеціальну кнопку `⋮` (MoreVertical). Вона розташована у правому нижньому куті, поруч із кнопкою редагування.

### Пункти меню
1. 📋 **Дублювати** — Створює копію файлу (з суфіксом `_copy.txt`) в тій самій вкладці. Конфігурації (strikethrough/colors) також копіюються.
2. 🗂️ **Перемістити** — Відкриває Sub-меню або Modal зі списком інших вкладок (папок). При виборі файл **фізично** копіюється в нову папку і видаляється зі старої. Конфігурація (записи в `cards`) також переноситься.
3. 🗑️ **Видалити** — Запит підтвердження ("Ви впевнені?"). Видаляє фізичний файл.
4. ↔️ **Рухати (Порядок)** — Sub-меню: "<- Ліворуч", "-> Праворуч" (впливає на `tab.order`).
5. ✏️ **Редагувати текст** — Аналогічно кліку на кнопку "Редагувати" (відкриває textarea).
6. ⚙️ **Налаштувати** — Відкриває Modal-вікно налаштувань самої картки.

## Налаштування картки (Modal)

Це вікно поверх усього екрану (Overlay) для правки конфігурації конкретної картки (тих даних, що лежать у `_hotpaste.json`).

### Поля Modal-вікна
- **`displayName` (Текст):** Назва картки (Placeholder = ім'я файлу).
- **`hotkey` (Текст 1 символ):** Призначення кастомної гарячої клавіші. Або вибір "Автоматично".
- **`icon` (Текст/Select):** Введення Emoji або назви іконки Lucide.
- **`color` (Color Picker):** Вибір кольору фону картки. Зручні пресети + кастом ввід HEX. Є кнопка "Скинути" (прозорий за замовчуванням).
- **`borderColor` (Color Picker):** Вибір кольору обводки картки.

**Кнопки:** "Скасувати" / "Зберегти". При збереженні записуємо в `_hotpaste.json` і картка миттєво оновлює UI.

*Технічна деталь:* Для Color Picker можна використовувати нативний `<input type="color">` або прості палітри кольорів-кнопок.

## Критерії прийняття

- [ ] Натискання '+' створює порожню картку в режимі редагування.
- [ ] При натисканні 'Зберегти' на новій картці, вона створюється на диску у поточній вкладці.
- [ ] Картки можна тягати мишкою, і їхній порядок зберігається після оновлення сторінки (F5).
- [ ] Сюрпризів із порядком немає, якщо додати новий файл через файловий менеджер ОС (він стає останнім).
- [ ] Правий клік та кнопка `⋮` у куті картки відкривають кастомне меню (однакове в обох випадках).
- [ ] Пункт 'Дублювати' створює копію (файл і UI одразу оновлюються).
- [ ] Пункт 'Видалити' видаляє файл назавжди (з попередженням).
- [ ] Пункт 'Перемістити' успішно переносить фізичний файл у папку іншої вкладки.
- [ ] Modal-вікно 'Налаштувати' змінює і зберігає (у json) `displayName`, `hotkey`, `icon`, `color`, `borderColor` вибраної картки.
