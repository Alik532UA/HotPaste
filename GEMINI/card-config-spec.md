## Сенс нововведень

### Проблема

Зараз картки — це "плоские" текстові файли без можливості кастомізації. Користувач не може:
- Давати зрозумілі назви карткам (показується ім'я файлу)
- Призначати кастомні гарячі клавіші
- Візуально розрізняти картки за кольором чи іконкою
- **Комбінувати схожі промти** — замість двох майже ідентичних промтів, що відрізняються 1-2 рядками, мати один з можливістю "вимкнути" непотрібні рядки

### Рішення

1. **Конфігурація карток** — JSON-файл `_hotpaste.json` в кожній папці-вкладці зі всіма налаштуваннями.
2. **Закреслення рядків** — інтерактивний фільтр: натиснув на рядок → він закреслюється і **не копіюється** при копіюванні. Натиснув ще раз → розкреслюється.

### Приклад: замість двох промтів — один

**Було (2 окремих файли):**

Файл `format-patch.txt`:
```
я вже зробив: `git format-patch --since="00:00:00" -o .temp/daily_patches/`
детально проаналізуй ці зміни та файли
...
```

Файл `diff-head.txt`:
```
я вже зробив: `git add -N . && git diff HEAD > .temp/.temp_diff.txt`
детально проаналізуй ці зміни та файли
...
```

**Стало (1 файл з toggle рядків):**

Файл `analyze-code.txt`:
```
я вже зробив: `git format-patch --since="00:00:00" -o .temp/daily_patches/`
я вже зробив: `git add -N . && git diff HEAD > .temp/.temp_diff.txt`
детально проаналізуй ці зміни та файли
оціни код "до" та "після" змін по шкалі від 0 до 100 балів
чи є зміни які погіршили код?
напиши звіт в новий .temp/temp_today_report.md файл
P.S. папка `.temp/` ігнорується налаштуваннями доступу, тож скористайся командою Get-Content
```

Користувач закреслює рядок 0 → при копіюванні він пропускається. Потрібен інший варіант → зняв закреслення з рядка 0, закреслив рядок 1.

## Архітектура зберігання

### Формат: `_hotpaste.json` (один файл на папку)

**Чому цей підхід:**

| Властивість | Значення |
|---|---|
| Separation of Concerns | ✅ Конфіг окремо від контенту, `.txt` файли залишаються чистими |
| Файлова чистота | ✅ 1 конфіг-файл на папку, а не ×2 файлів на картку |
| Масштабованість | ✅ Можна додати tab-level налаштування (колір вкладки, іконка, тощо) |
| Синхронізація | ✅ Файл на диску → працює з Git/OneDrive/Dropbox |

### Структура файлу

```json
{
  "tab": {
    "icon": "📂",
    "color": null,
    "displayName": null
  },
  "cards": {
    "analyze-code.txt": {
      "displayName": "Аналіз коду",
      "hotkey": "z",
      "icon": "🔍",
      "color": null,
      "borderColor": null,
      "strikethrough": [0, 1]
    },
    "greeting.txt": {
      "displayName": null,
      "hotkey": null,
      "icon": "👋",
      "color": "#1a2b3c",
      "borderColor": "#00d2ff",
      "strikethrough": []
    }
  }
}
```

### Правила роботи з файлом

- Якщо `_hotpaste.json` **не існує** — все працює як зараз (дефолтні значення).
- Якщо для картки **немає запису** в `cards` — дефолтні значення.
- `null` означає "використати дефолт".
- **Debounced запис**: при зміні strikethrough → оновлення in-memory миттєво, запис на диск через 500мс після останньої зміни.
- Файл створюється **автоматично** при першій зміні (перший toggle strikethrough, перше призначення hotkey, тощо).

### Розташування на диску

```
prompts/
├── 01_General/
│   ├── _hotpaste.json        ← конфіг для вкладки "General"
│   ├── greeting.txt
│   ├── summary.txt
│   └── translate.txt
├── 02_Code/
│   ├── _hotpaste.json        ← конфіг для вкладки "Code"
│   ├── analyze-code.txt
│   └── refactor.txt
```

## Атрибути картки

| Атрибут | Тип | Дефолт | Опис |
|---|---|---|---|
| `displayName` | `string \| null` | Ім'я файлу без розширення та цифрового префіксу | Назва, що відображається на картці |
| `hotkey` | `string \| null` | Автоматичне QWERTY призначення | Кастомна гаряча клавіша |
| `icon` | `string \| null` | Немає | Emoji або назва Lucide іконки |
| `color` | `string \| null` | Прозорий (дефолтний `--color-card-bg`) | Колір фону картки (CSS color) |
| `borderColor` | `string \| null` | Прозорий (дефолтний `--color-border`) | Колір обводки картки (CSS color) |
| `strikethrough` | `number[]` | `[]` | Індекси закреслених рядків (0-based) |

### Пріоритет гарячих клавіш

1. Спочатку призначаються **кастомні** hotkey (з `_hotpaste.json`).
2. Потім автоматичні QWERTY клавіші заповнюють решту, **пропускаючи** вже зайняті літери.

**Приклад:** Якщо картка `review.txt` має `"hotkey": "a"`, то автоматичне призначення пропустить `a` і видасть: `q, w, e, ...` (без `a`).

## Закреслення рядків

### Концепція

Кожна картка має **per-line toggle**: будь-який рядок можна "закреслити" (strikethrough). Закреслені рядки:

- **Відображаються** в UI з візуальним закресленням (`text-decoration: line-through`, приглушений колір)
- **НЕ копіюються** при натисканні hotkey або кліку в "зону дії"
- Стан зберігається в `_hotpaste.json` → `cards[filename].strikethrough` як масив індексів рядків

### Зберігання стану

- Ідентифікація: за **індексом рядка** (0-based)
- При **редагуванні** картки (зміна контенту): автоматичний перерахунок індексів
  - Видалений рядок → його індекс вилучається, наступні зсуваються
  - Доданий рядок → індекси після нього зсуваються +1
  - Зміна тексту рядка без зміни кількості рядків → індекси не змінюються

### UI відображення закресленого рядка

```
┌──────────────────────────────────────────┐
│ 🔍 Аналіз коду                     [z]  │
│                                          │
│ ~~я вже зробив: `git format-patch...`~~  │  ← закреслений (dim + line-through)
│ я вже зробив: `git add -N . && git...`   │  ← звичайний
│ детально проаналізуй ці зміни та файли   │
│ оціни код "до" та "після" змін...        │
│ чи є зміни які погіршили код?            │
│ напиши звіт в новий .temp/...            │
│ P.S. папка `.temp/` ігнорується...       │
└──────────────────────────────────────────┘
```

## Зони взаємодії (Click Zones)

### Архітектура зон

Картка має **два шари** + кнопка (`edit-indicator`) обробки кліків:

```
┌──────────────────────────────────────────┐
│                                          │
│   ЛІВА 20%            │  ПРАВА 80%       │
│   (Закреслення)       │  (Зона дії)      │
│                       │                  │
│   Toggle рядка        │  Копіювання      │
│   (strike/unstrike)   │                  │
│                       │              [B] │
└──────────────────────────────────────────┘
```
[B] це кнопка редагування `edit-indicator`

**Реалізація:**

1. **Ліва 50% картки** — обробляє закреслення рядків. Визначає на який рядок натиснули → toggle strikethrough.
2. **Права 50% картки** — зона копіювання (перехоплює клік → копіює картку в буфер обміну).
3. **Кнопка редагування** — з'являється у нижньому правому куті при наведенні, відкриває inline-редактор.

**Чому саме так:** Більше немає окремих глобальних режимів системи ("Копіювання" чи "Редагування"). Картка завжди готова і до копіювання, і до фільтрації рядків, і до редагування (через кнопку).

### Закреслення у різних виглядах карток

| Вигляд | Toggle strikethrough |
|---|---|
| **Короткий** | Працює тільки на **видимих** рядках |
| **Повний** | Працює на **всіх** рядках |

### Hotkey копіювання

При натисканні гарячої клавіші картки (QWERTY) — копіюється текст **без закреслених рядків** (аналогічно кліку на ліву зону).

## Критерії прийняття

### AC 1: Конфігурація карток

- [ ] Картка з `displayName` в `_hotpaste.json` показує кастомну назву
- [ ] Картка з `hotkey: "z"` реагує на клавішу `z`, автоматичне призначення пропускає `z`
- [ ] Картка з `icon: "🔍"` показує іконку в заголовку
- [ ] Картка з `color: "#1a2b3c"` має відповідний фон
- [ ] Картка з `borderColor: "#00ff88"` має відповідну обводку
- [ ] Якщо `_hotpaste.json` відсутній — все працює як раніше (дефолти)

- [ ] Стан зберігається в `_hotpaste.json` → `cards[filename].strikethrough`
- [ ] При перезавантаженні сторінки — закреслення відновлюються з файлу

### AC 3: Копіювання з фільтрацією
